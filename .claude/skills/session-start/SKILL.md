---
name: session-start
description: Run at the start of every session to restore context before proposing any work.
---

1. Read `journal/SESSIONS.md` — top entry only (most recent session).
2. Read `journal/BACKLOG.md` — identify what is pending and what phase is active.
3. Read `journal/DECISIONS.md` — note any constraints that affect today's work.
4. Summarize to the user: what was done last session, what is next, any open questions or blockers.
5. Wait for the user to confirm the context or redirect before proposing or starting any work.
